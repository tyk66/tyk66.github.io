二、单篇文章核心操作

操作类型 完整命令 简写命令 详细说明
创建普通文章 hexo new "文章标题" hexo n "文章标题" 在 source/_posts 生成带默认配置的 .md 文件
创建纯文本文章 hexo new post "纯文本标题" --draft hexo n post "纯文本标题" -d 生成无模板的极简 .md 文件
创建独立页面 hexo new page "页面名" hexo n page "页面名" 生成独立页面（如 about 关于页），路径 source/页面名/index.md
本地预览 hexo server hexo s 启动本地服务器，访问 http://localhost:4000 查看效果
预览自动刷新 hexo server -w hexo s -w 监听文件变化，编辑后刷新浏览器即可看最新内容
自定义预览端口 hexo server -p 5000 hexo s -p 5000 4000 端口被占用时使用，访问 http://localhost:5000
编辑后部署上线 hexo clean && hexo generate && hexo deploy hexo cl && hexo g && hexo d 清理缓存 → 生成静态文件 → 部署到 GitHub Pages
手动删除文章 直接删除 source/_posts 下的目标 .md 文件，再执行部署命令 - 新手推荐，直观不易出错
终端删除文章（Windows） rm source/_posts/目标文章.md - 需准确输入文件名，注意中文和空格

六、新手避坑提醒

  • 命令路径错误：所有命令必须在博客根目录执行，否则会提示 hexo 不是内部命令 或文件找不到。

  • 中文标题问题：创建中文标题文章时，必须给标题加引号，例如 hexo n "我的第一篇博客"，否则终端识别出错。

  • 部署后无变化：优先执行 hexo clean 清理缓存，再重新部署；若仍无变化，等待 5 分钟（GitHub 同步延迟）。

  • 端口占用问题：4000 端口被占用时，用 hexo s -p 5000 切换端口，或关闭占用 4000 端口的程序（如微信开发者工具、其他本地服务器）。
